GIT/Git Flow Task

1) Кратко опишите что делает каждая команда в GIT
git init - инициализация нового репозитория, начало отслеживания
git clone - создание локально копии проекта из удаленного репозитория
git add - подготовка изменения в коде / файле проекта, первый этап создания моментального снимка проекта
git commit - создание снимка / отпечатка проекта на текущий момент, второй этап отслеживания изменений
git push - обновление удаленного репозитория изменениями в локальной ветке
git pull - применение к локальной ветке изменений из удаленного репозитория ("стягивание" текущей версии проекта)
git branch - вывод ветвей локальной разработки
git checkout - переключение между ветками, созданными git branch
git merge - слияние веток в одну, изменения указанной ветки вливаются в текущую ветку
git rebase - объединение двух веток, переписывает историю ветки
git cherry-pick - выборочное применение коммитов к текущей ветке head, можно выбрать коммит из одной ветки и применить его к другой
git status - отображение текущего состояния каталога и файлов в нем
git log - просмотр истории коммитов, начиня с самого свежего
git reset - перемещение указателя head и указателя ветки на заданный коммит
git revert - отменяет изменения в ветке созданием нового коммита с отменой предыдущих изменений
git stash - откладывание изменений, сделанных в рабочей копии для возможности применить их позже
git diff - выявление разницы между двумя ветками / коммитами / файлами и т.п.
